# BagsSignal

BagsSignal is a creator intelligence dashboard for the Bags ecosystem. It helps Bags token creators and reviewers understand creator fee revenue, claimable earnings, token momentum, and AI-ready market signals without needing to manually inspect every token page or wallet.

The core problem BagsSignal solves is visibility. Most creators can see that Bags enables fee revenue, but it is still hard to answer practical questions such as:

- Which tokens are producing meaningful creator revenue?
- Which tokens have claimable fees right now?
- Which tokens are gaining momentum from recent trading activity?
- What should a creator do next: claim, promote, monitor, or wait?
- How can reviewers explore the product when their own wallet has no Bags activity?

BagsSignal combines Bags API data, Solana on-chain token metadata, Bitquery trading data, local SQLite caching, and AI-style recommendation layers into one focused product experience.

## Features

- Creator dashboard for wallet-level Bags fee revenue.
- Sample mode for reviewers without Bags creator activity.
- Claim history view with sample history support.
- Bags creator leaderboard backed by local SQLite data.
- Token cards with Bags token page links.
- AI Revenue Advisor for wallet-level creator actions.
- AI Market Read and per-token AI Signal on the leaderboard.
- Composite ranking formula that blends revenue, trading, claimable fees, activity, and momentum.
- Local data sync pipeline using Bags API + Bitquery + SQLite.
- Configurable AI provider layer for Anthropic, OpenAI, and OpenAI-compatible APIs.

## Product Walkthrough

### Home

The landing page introduces BagsSignal as a creator revenue and market signal workspace for Bags. Users can connect a wallet from the top-right wallet button or open the live leaderboard directly without connecting a wallet.

This is important for hackathon judging because most users will not have a wallet with Bags creator revenue. The project therefore supports both a real connected-wallet flow and a sample exploration flow.

### Dashboard

The Dashboard is the personal creator view. When a wallet has Bags activity, it shows:

- Total claimable creator fees.
- Lifetime creator earnings.
- Total fees generated by tracked tokens.
- Creator token cards.
- Revenue trend visualization.
- AI Revenue Advisor.

Each token card includes:

- Token image from metadata when available.
- Symbol and token name.
- Contract address preview.
- Claimable balance.
- Lifetime earned amount.
- Graduation or bonding curve status.
- Claim action.
- Direct `Bags` link to the token page.

If the connected wallet has no Bags creator data, the Dashboard shows a clear empty state. The user can enter sample mode to view a realistic creator dashboard without pretending the data belongs to their wallet.

#### AI Revenue Advisor

The Dashboard AI Revenue Advisor is designed as a creator action panel rather than a raw analytics report.

It focuses on three practical questions:

- `Claim Timing`: when it makes sense to claim fees.
- `Top Token Focus`: which token deserves creator attention.
- `Growth Signal`: what market or revenue behavior is worth watching.

For demo reliability, the current UI includes a built-in advisor playbook with 20 recommendation sets. When the user clicks `Refresh`, BagsSignal randomly selects one recommendation set and simulates model-style streaming output. This keeps the product experience available even when an external AI API key is missing or rate-limited.

The codebase also includes a configurable AI provider layer that can call Anthropic, OpenAI, or OpenAI-compatible APIs for future generated insights.

### History

The History page shows fee claim history for a connected wallet when claim events are available.

For wallets without claim history, the page offers sample mode. Sample history uses real token and revenue context from the local leaderboard cache, but it does not fake real transaction hashes. In sample mode, the final column changes from `Transaction` to `Token Page`, and each row links to `Open on Bags`.

This keeps the UI honest: real wallet history can show real claim transactions, while sample mode shows a realistic walkthrough without pretending generated rows are on-chain transactions.

### Leaderboard

The Leaderboard ranks active Bags token creators by a composite score. It is not presented as a full historical index of every Bags token. Instead, it tracks an active token universe built from recent launches, active Bags mints, trading activity, and enriched Bags creator data.

The page includes:

- Live/Demo data indicator.
- Total creators ranked.
- Total fees generated.
- Highest single earner.
- Tracking logic explanation.
- Top 3 podium.
- AI Market Read.
- Searchable token table.
- Per-token AI Signal expansion.
- Token contract address.
- Creator social handle and platform icon when available.
- Direct Bags page link.

#### Leaderboard Ranking Formula

Each token receives normalized component scores, then BagsSignal calculates:

```text
Score = 40% creator earnings
      + 25% 24h volume
      + 15% claimable fees
      + 10% trade activity
      + 10% momentum
```

The implementation uses log normalization for revenue, claimable fees, volume, and trade count so that extremely large values do not completely dominate the ranking.

Momentum is calculated from:

- 24h trade activity.
- 24h volume strength.
- Launch recency.

In code, momentum is weighted as:

```text
Momentum = 55% trade score + 20% volume score + 25% recency score
```

This makes the leaderboard favor tokens that combine creator revenue with current market activity, rather than ranking only by lifetime totals.

#### AI Market Read

The AI Market Read summarizes the visible market pattern across the current leaderboard page. It is intentionally short and opinionated, focusing on whether the current Bags token set is momentum-led, revenue-led, concentrated, or still waiting for stronger liquidity confirmation.

#### Per-Token AI Signal

Each leaderboard token can expose an AI Signal card. The signal is generated during the leaderboard sync and stored in SQLite.

The sync stores AI-ready features such as:

- 1h, 6h, 24h, and 7d volume.
- 1h, 6h, 24h, and 7d trade count.
- Volume growth ratios.
- Trade velocity.
- Revenue velocity.
- Metadata image availability.
- Creator social signal availability.
- Low-liquidity risk.
- High-momentum flags.
- AI readiness score.

The current recommendation engine is rule-based and deterministic. It produces:

- Recommendation type: momentum, revenue, risk, hidden gem, or creator action.
- Stance: bullish, watch, cautious, or risk.
- Signal strength: Strong Signal, Developing Signal, or Needs More Data.
- Insight.
- Recommended action.
- Evidence metrics.

This design makes recommendations stable for demos while keeping the data model ready for future LLM-generated recommendations.

## Data Pipeline

BagsSignal uses a local SQLite-backed data pipeline for the leaderboard.

### 1. Token Discovery

The sync discovers Bags-related tokens from multiple sources:

- Bitquery token supply updates signed by the Bags launch signer.
- Bitquery program launch events from the Meteora DBC program.
- Bitquery active trade tokens whose mint ends with `BAGS`.
- Bags token launch feed as a fallback discovery source.

### 2. Bags API Enrichment

For each candidate token, BagsSignal enriches data using Bags API endpoints and related Bags pages:

- Claimable positions by wallet.
- Token lifetime fees.
- Token creator information.
- Token launch feed.
- Claim events.
- Claim transaction preparation.
- Bags token page metadata fallback.

The project uses the public Bags API base:

```text
https://public-api-v2.bags.fm/api/v1
```

Requests include the configured `BAGS_API_KEY`.

### 3. On-Chain Metadata

Token images are resolved from Solana token metadata when possible. BagsSignal reads the token Metaplex metadata PDA through `@solana/web3.js`, extracts the metadata URI, then resolves image information from the token metadata JSON.

If the on-chain metadata image is unavailable, the app falls back to the Bags token page Open Graph image when possible.

### 4. Trading Metrics

Bitquery is used to fetch trading activity for Bags mints:

- 1h volume and trades.
- 6h volume and trades.
- 24h volume and trades.
- 7d volume and trades.
- Last trade timestamp.

These metrics power the leaderboard score, momentum score, AI feature table, and AI Signal evidence.

### 5. SQLite Cache

The enriched rows are stored locally in SQLite:

- `leaderboard_entries`
- `token_ai_features`
- `token_ai_recommendations`
- `leaderboard_sync_runs`
- `app_state`

The UI reads leaderboard data from SQLite instead of querying Bags and Bitquery on every page load. This keeps the app fast, protects API usage, and makes the demo more resilient.

## Tech Stack

- Next.js 14 App Router.
- React 18.
- TypeScript.
- Tailwind CSS.
- SWR for client-side data fetching.

```bash

NEXT_PUBLIC_SOLANA_RPC_URL=https://api.mainnet-beta.solana.com
NEXT_PUBLIC_SOLANA_NETWORK=mainnet-beta
NEXT_PUBLIC_APP_URL=http://localhost:3000

BITQUERY_ACCESS_TOKEN=your_bitquery_access_token
BITQUERY_GRAPHQL_URL=https://streaming.bitquery.io/graphql
LEADERBOARD_SQLITE_PATH=data/bagssignal.sqlite
LEADERBOARD_SYNC_LIMIT=100

AI_PROVIDER=anthropic

OPENAI_COMPATIBLE_BASE_URL=
OPENAI_COMPATIBLE_MODEL=
```

## Local Development

Install dependencies:

```bash
npm install
```

Run the development server:

```bash
npm run dev
```

Open:

```text
http://localhost:3000
```

Run type checking:

```bash
npm run typecheck
```

## Manual Leaderboard Sync

The project is designed to support manual leaderboard updates so API keys are not consumed unexpectedly.

When the dev server is running, trigger a sync with:

```bash
curl -X POST http://localhost:3000/api/leaderboard/sync
```

Check sync status:

```bash
curl http://localhost:3000/api/leaderboard/status
```

## Project Philosophy

BagsSignal is not just a fee dashboard. It is a signal layer for Bags creators.

The product is built around a simple idea: creators need to know what to do next. Raw token data is useful, but creator revenue decisions require context: claimable fees, lifetime earnings, trading momentum, token metadata quality, social creator signals, and risk flags.

BagsSignal turns those inputs into a practical interface for creators, reviewers, and market watchers.
